Detailed Description

The FeederPublisher class provides an abstraction ontologenius feeder(insert) ROS topic.

Working in a closed world can be interesting, but with ontologenius, you can also choose to work in an open world by adding and modifying the agent's knowledge base during its operation. The feeder publisher is used to insert and delete knowledge dynamically. The feeding process is asynchronous and therefore does not guarantee any response time. It still provides a function to synchronize if you have applications where you have to query the ontology right after having modified it.

FeederPublisher::FeederPublisher(ros::NodeHandle* n, const std::string& name)

Constructs an FeederPublisher with a pointer to a NodeHandle n.

Can be used in a multi-ontology mode by specifying the name of the ontology name. For classic use, name should be defined as "".

FeederPublisher::addProperty(const std::string& from, const std::string& property, const std::string& on)

Adds the fact that from is linked with on by the property property.

At least from or on must be already known to the system. If one of them is unknown, it will be automatically created. The property can be unknow before calling this function.

FeederPublisher::addProperty(const std::string& from, const std::string& property, const std::string& type, const std::string& value)

Adds the fact that from is linked to the data value of type type by the property property.

from must be already known to the system. The property can be unknow before calling this function.

FeederPublisher::addInheritage(const std::string& from, const std::string& on)

Adds the inheratage : from is a on. from and on could by a class, an individual or a property.

At least from or on must be already known to the system. If one of them is unknown, it will be automatically created.

FeederPublisher::addLanguage(const std::string& from, const std::string& lang, const std::string& name)

from must be already known to the system.

FeederPublisher::removeConcept(const std::string& from)

size_t FeederPublisher::getNumSubscribers()

Returns the number of subscribers that are currently connected to the internal ROS publisher.

bool FeederPublisher::commit(int32_t& timeout = -1)

Waits until all changes have been applied.

The default parameter timeout is the expiration time in milliseconds. The default value -1 represents an infinite wait.

Returns false if the function returns on a timeout.
